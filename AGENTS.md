# Codex作業ルール — timebox-app

このリポジトリは、iPad Safari・ホーム画面追加版・PCブラウザで使用する個人用タイムボックスWebアプリ `timebox-app` の専用リポジトリである。

Codexは作業前に、この `AGENTS.md`、`UI_STYLE_GUIDE.md`、`TIMEBOX_SPEC.md`、`README.md` を確認すること。

## リポジトリの独立性

- このリポジトリだけを変更対象とする。
- 次の既存リポジトリや、そのローカルフォルダーには変更を加えない。
  - `sazanami3691/memo-app`
  - `sazanami3691/rpg-maker-mz-plugins`
- 別リポジトリのコードや設定を、明示的な許可なくコピー・変更・コミットしない。

## 作業ブランチ

- Codexが使用する作業ブランチは `codex-work` とする。
- 作業開始前に `git branch --show-current` と `git status` を確認する。
- `codex-work` 以外の場合はファイルを変更せず、現在の状態を報告する。
- `main` には直接コミット・プッシュしない。

## 作業前の確認

- `git status --short`、必要に応じて `git diff` を確認する。
- ユーザーが作成した未コミット変更を削除・上書き・破棄しない。
- 関係のないファイルを変更しない。
- 仕様に不明点があっても、既存文書から安全に判断できる範囲は進める。重大な選択だけ報告する。

## 実装前の説明

実装を始める前に、次を簡潔に説明する。

- 実装負荷
- 変更対象
- iPad特有のリスク
- IndexedDBなど保存データへの影響
- 既存機能との競合可能性
- 一度に実装してよいか、安全な単位へ分けるべきか

重大な追加判断が不要なら、説明後そのまま実装を続ける。

## 技術方針

- HTML、CSS、JavaScriptを第一候補とする。
- iPad Safari、ホーム画面追加版、PCブラウザで利用できる構成にする。
- サーバー、ログイン、クラウド同期を前提にしない。
- 予定・履歴・実行中タイマーはIndexedDBを中心に保存する。
- 小さな表示設定などにLocalStorageを使用してよいが、保存責務を明確に分ける。
- 外部ライブラリを追加する場合は、必要性、オフライン対応、保守負荷を説明する。安易に大きな依存を増やさない。
- 第1段階では基本版に集中し、PWA・通知など第2段階の機能を先回りして複雑に実装しない。

## タイマー実装の必須原則

- 単純に1秒ずつ残り時間を減らす値だけを正としない。
- 実行中は開始時刻、終了予定時刻、状態、残り時間など、復帰に必要な情報を永続化する。
- 表示更新時、ページ復帰時、再読み込み時には現在時刻との差から残り時間を再計算する。
- バックグラウンドや画面消灯でJavaScriptの更新が止まっても、復帰後に正しい状態へ戻す。
- 一時停止中は経過時間を加算せず、保存した残り時間を維持する。
- 同時に動作するタイマーは1つだけとする。
- 実行中・一時停止中・終了待ちの状態遷移を明示的に管理する。
- 端末の時計変更や不正な保存値で破綻しないよう、入力値と復元値を検証する。

## 保存データと互換性

- 保存データにはスキーマバージョンを持たせる。
- 予定、履歴、設定、実行中タイマーの責務を分ける。
- 各データには衝突しにくい一意IDを使用する。
- 日付と日時の保存形式を統一し、ローカル日付と絶対時刻を混同しない。
- 保存済みデータを破壊する変更は避ける。
- スキーマ変更時は移行処理、失敗時の安全な停止、既存データ維持を考慮する。
- テスト用データや個人の予定内容をコミットしない。

## UIと操作

- UIの新規作成・変更前に `UI_STYLE_GUIDE.md` を確認する。
- iPad縦画面を優先しつつ、横画面とPCでも破綻させない。
- タッチ対象は原則44px以上にする。
- hoverだけに依存しない。
- セーフエリア、ソフトウェアキーボード、スクロール、モーダルの画面外はみ出しを確認する。
- PCではマウス操作とキーボードフォーカスも確認する。

## 実装段階

- `TIMEBOX_SPEC.md` の「第1段階」と「第2段階」を混同しない。
- ユーザーから第1段階の依頼を受けた場合、第2段階の通知、PWA、ドラッグ並べ替えなどを勝手に追加しない。
- 将来拡張できるデータ設計は行ってよいが、未依頼のUIや動作は公開しない。

## 検証

変更後、利用可能な範囲で次を行う。

- JavaScript構文チェック
- HTML/CSSの基本的な検証
- 主要ロジックのテスト
- IndexedDBの保存・再読込確認
- タイマーの開始、一時停止、再開、完了、スキップ、延長
- ページ再読み込み後の時刻再計算
- iPad相当の狭い縦画面、横画面、PC幅での表示確認
- タッチ操作とマウス操作の確認

自動テストで確認できないiPad固有挙動は、未確認事項として明記する。

## 自動コミットとプッシュ

- 実装と確認に問題がなければ、`git status` と `git diff` で最終確認する。
- 今回変更した関係ファイルだけを個別に `git add` する。
- 内容が分かる簡潔なコミットメッセージでコミットする。
- `origin/codex-work` へプッシュする。
- 変更がない場合は空コミットしない。
- 作業完了時に、変更ファイル、実行した確認、未確認事項、コミットID、プッシュ先を報告する。

## 禁止事項

- `main` への直接コミット・プッシュ
- force push
- 無断でのブランチ削除
- 無断でのマージまたはPull Request作成
- `git reset --hard` などによるユーザー変更の破棄
- 無関係なファイルの一括追加
- 秘密情報、`.env`、ログ、キャッシュ、ビルド生成物、ユーザーデータのコミット
- 依頼のない既存リポジトリへの変更

## 問題発生時

Git競合、認証エラー、通信エラー、保存データ移行失敗、テスト失敗が発生した場合は、無理に続けず状況を報告する。対象や安全な復旧方法が明確でない場合、推測でリセット・復元・削除を実行しない。
