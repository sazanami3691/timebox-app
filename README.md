# timebox-app

iPadを中心に、PCブラウザでも利用できる個人用タイムボックスWebアプリです。

ポモドーロ専用ではなく、作業名と時間を自由に設定し、その日の予定を「時刻指定の簡易時刻表」と「時刻未指定のタスク一覧」で管理できます。サーバー、ログイン、外部API、実行時の外部CDN依存はありません。

公開予定URL：<https://sazanami3691.github.io/timebox-app/>

## 現在実装済み

### 第1段階：基本版

- 今日・未来日の時間指定／時刻指定予定の登録、編集、複製
- 作業と休憩の区別、時刻指定予定の重複防止
- 登録順の一覧、完了・スキップ済み欄、過去日の閲覧
- 取り消し可能な予定削除と前日の予定一括コピー
- タイマーの開始、一時停止、再開、完了、スキップ、5分延長
- 早期開始、遅延開始、終了待ち、超過時間の実績記録
- タイマーを使わない手動完了
- ページ再読み込み・画面復帰時のタイマー再計算
- 日別の完了数、スキップ数、合計作業時間、達成率、休憩実績
- iPad縦画面優先のレスポンシブUIとキーボード操作

タイマーは1秒ごとの減算値を正とせず、終了予定時刻と現在時刻の差から表示を再計算します。一時停止中は保存した残り時間を維持し、同時に動作できるタイマーは1つだけです。

### 第2段階A：PWA・公開準備

- Web App ManifestとiPad／PWA用アイコン
- Service Workerによるアプリシェルの事前キャッシュ
- 初回オンラインアクセス後のオフライン起動
- 新バージョン検出とユーザー操作による更新
- 実行中・一時停止中・終了待ちタイマーがある場合の更新保留
- オンライン／オフライン状態とオフライン準備状態の表示
- 設定・アプリ情報画面とホーム画面追加案内
- GitHub Pagesのプロジェクトサブパス対応と`.nojekyll`

現在のアプリバージョンは`1.1.0`です。新しいService Workerを検出しても自動更新・自動再読み込みは行いません。「更新する」を押した場合だけ適用し、`controllerchange`後に1度だけ再読み込みします。

## 保存先とデータ維持

予定、履歴、現在のタイマー、スキーマ情報はブラウザのIndexedDB `timebox-app` に保存します。Service Workerは静的なアプリファイルだけをキャッシュし、IndexedDBやLocalStorageには触れません。そのためアプリ更新後も既存データを維持します。

Safariの履歴・Webサイトデータを削除した場合や、OSがサイトデータを消去した場合は、保存内容も失われる可能性があります。JSONバックアップはまだ未実装のため、重要な予定の唯一の保管場所にはしないでください。

## オフライン利用とホーム画面追加

初回アクセス時にはキャッシュがないため、オフラインでは起動できません。オンライン状態で一度アプリを開き、設定画面に「オフライン準備：準備完了」と表示された後からオフライン起動できます。

iPadではGitHub Pages公開後にSafariで開き、次の手順で追加します。

1. Safariの共有ボタンを開く。
2. 「ホーム画面に追加」を選ぶ。
3. 追加後、ホーム画面のTimeboxアイコンから起動する。

## ローカル起動とService Worker確認

Service WorkerとES Modulesを使用するため、ファイルを直接開かずローカルHTTPサーバー経由で起動してください。`127.0.0.1`はブラウザで安全なローカルコンテキストとして扱われます。

```powershell
python -m http.server 8000 --bind 127.0.0.1
```

PCブラウザで`http://127.0.0.1:8000/`を開きます。

1. 開発者ツールのConsoleにエラーがないことを確認する。
2. Application（またはStorage）パネルでManifestを確認する。
3. Service Workersで`sw.js`がactivatedになっていることを確認する。
4. Cache Storageに`timebox-app-shell-1.1.0`があることを確認する。
5. 設定・アプリ情報画面でオフライン準備完了を確認する。
6. 開発者ツールのOfflineを有効にして再読み込みし、予定・履歴が残ることを確認する。

同じオリジンを使い続ける必要があるため、確認時はホスト名とポートを変更しないでください。Service Workerの更新確認では、ブラウザキャッシュではなく新しいWorkerとバージョン定義を取得します。

## テスト

外部テストライブラリは使用していません。

```powershell
npm test
```

アイコンは標準の`System.Drawing`だけを使用する次のスクリプトで再生成できます。

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\tools\generate-icons.ps1
```

## GitHub Pagesの手動設定

Codexはリポジトリ設定を変更しません。実装を確認した後、GitHub上で次の操作を行います。

1. `sazanami3691/timebox-app`の「Settings」を開く。
2. 左側の「Pages」を開く。
3. Sourceで「Deploy from a branch」を選ぶ。
4. Branchを`codex-work`、Folderを`/(root)`にする。
5. 「Save」を押し、公開完了後に予定URLを開く。

最終確認後は公開元ブランチを`main`へ切り替える予定です。CodexによるmainへのマージやPages設定変更は行いません。

## 今後追加予定・既知の制限

- Web通知、タイマー終了音、予定時刻通知は未実装です。
- Wake Lockによる画面点灯維持は未実装です。
- 長押しドラッグ並べ替え、全期間検索は未実装です。
- JSONバックアップ、過去履歴編集は未実装です。
- 保存データは使用中のブラウザ・オリジン内だけに存在し、端末間同期はありません。
- アプリを完全に閉じている間の通知はありません。復帰時に現在時刻から終了待ちへ復元します。
- 過去日の予定と履歴は閲覧専用です。
- GitHub Pages公開後のiPad Safari・ホーム画面追加状態での実機確認が必要です。

詳細仕様は[`TIMEBOX_SPEC.md`](./TIMEBOX_SPEC.md)、UI基準は[`UI_STYLE_GUIDE.md`](./UI_STYLE_GUIDE.md)、開発ルールは[`AGENTS.md`](./AGENTS.md)を参照してください。
