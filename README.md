# timebox-app

iPadを中心に、PCブラウザでも利用できる個人用タイムボックスWebアプリです。

ポモドーロ専用ではなく、作業名と時間を自由に設定し、その日の予定を「時刻指定の簡易時刻表」と「時刻未指定のタスク一覧」で管理できます。サーバー、ログイン、外部API、実行時の外部CDN依存はありません。

## 現在実装済み：第1段階（基本版）

- 今日・未来日の時間指定／時刻指定予定の登録、編集、複製
- 作業と休憩の区別、時刻指定予定の重複防止
- 登録順の一覧、完了・スキップ済み欄、過去日の閲覧
- 取り消し可能な予定削除
- 前日の予定を選択中の日へ一括コピー
- タイマーの開始、一時停止、再開、完了、スキップ、5分延長
- 早期開始と、遅延開始時の「予定時間すべて／本来の終了まで」選択
- 終了待ち、超過表示、予定時間のみ／超過込みの実績記録
- タイマーを使わない手動完了
- 実行中予定の作業名・メモ編集
- ページ再読み込み、画面復帰時のタイマー再計算
- 日別の作業完了数、スキップ数、合計作業時間、達成率、休憩実績
- iPad縦画面優先のレスポンシブUIとキーボード操作

タイマーは1秒ごとの減算値を正とせず、終了予定時刻と現在時刻の差から表示を再計算します。一時停止中は保存した残り時間を維持し、同時に動作できるタイマーは1つだけです。

## 保存先

予定、履歴、現在のタイマー、スキーマ情報はブラウザのIndexedDB `timebox-app` に保存します。サーバーや別端末へは送信しません。

Safariの履歴・Webサイトデータを削除した場合や、OSがサイトデータを消去した場合は、保存内容も失われる可能性があります。現時点ではJSONバックアップ機能が未実装のため、重要な予定の唯一の保管場所にはしないでください。

## ローカル起動

ES ModulesとIndexedDBを使用するため、ファイルを直接開かずローカルHTTPサーバー経由で起動してください。

Pythonがある場合：

```powershell
python -m http.server 8000 --bind 127.0.0.1
```

その後、PCブラウザで `http://127.0.0.1:8000/` を開きます。

## PCでの確認

1. 上記のローカルサーバーを起動する。
2. ブラウザの開発者ツールを開き、Consoleにエラーがないことを確認する。
3. 予定を登録し、再読み込み後も残ることを確認する。
4. タイマーの開始、一時停止、再開、5分延長、完了、スキップを確認する。
5. 開発者ツールのレスポンシブ表示で、iPad縦相当・横相当とPC幅を確認する。
6. Application（またはStorage）パネルでIndexedDBの4ストアを確認する。

ロジックテストはNode.js標準機能だけで実行できます。

```powershell
npm test
```

## 第2段階で追加予定

- Web通知と終了音
- PWA、Service Worker、オフラインキャッシュ、ホーム画面追加対応
- Wake Lockによる画面点灯維持
- 専用ハンドルによる長押しドラッグ並べ替え
- 全期間検索
- JSONバックアップ
- 過去履歴の編集
- アプリ更新検出
- GitHub Pages公開

未実装機能は現在の画面で利用できません。iPad実機での最終確認はGitHub Pages公開後に行う予定です。

## 既知の制限

- 保存データは使用中のブラウザ・オリジン内だけに存在し、端末間同期はありません。
- JSONバックアップと復元は未実装です。
- 通知、終了音、画面点灯維持、オフライン利用には対応していません。
- アプリを完全に閉じている間は通知しません。復帰時に現在時刻から終了待ちへ復元します。
- 過去日の予定と履歴は閲覧専用です。
- 並べ替えは登録順のみで、ドラッグ操作は未実装です。
- iPad Safari・ホーム画面追加状態での実機確認は未実施です。

- 作業ブランチ：`codex-work`
- `main`へ直接コミット・プッシュしない
- force push、無断マージ、無断のPull Request作成をしない
- テスト成功後、関係ファイルだけを `origin/codex-work` へプッシュする
