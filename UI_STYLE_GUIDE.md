# timebox-app UIスタイルガイド

この文書は、iPadを中心に使用する個人用タイムボックスWebアプリ `timebox-app` のUI基準である。

## 基本方針

- 黒または濃いグレーの背景を使用する。
- 黒系のウィンドウと、白または薄いグレーの枠線を基準にする。
- 明るい文字を使用し、長時間見ても疲れにくいコントラストにする。
- 装飾より視認性、状態の分かりやすさ、押しやすさを優先する。
- 重要な操作だけアクセントカラーを使用する。
- 既存の自作メモアプリに近い「黒いウィンドウ＋白い枠」の印象を基準にするが、別リポジトリのコードはコピーしない。

## 対象端末

優先順位は次のとおり。

1. iPadの縦画面
2. iPadの横画面
3. PCブラウザ

縦画面を基準にするが、横画面とPCで機能や操作が欠けないようにする。

```css
:root {
  --ui-bg: #0d1117;
  --ui-header: #0f141b;
  --ui-panel: #171d25;
  --ui-control: #1b232d;
  --ui-border: #34404d;
  --ui-border-strong: #d5d9e2;
  --ui-text: #f1f3f5;
  --ui-text-muted: #aeb7c2;
  --ui-accent: #3f8f55;
  --ui-warning: #b58a3a;
  --ui-danger: #b84a4a;
  --ui-focus: #78a9ff;
  --ui-radius-sm: 8px;
  --ui-radius-md: 12px;
  --ui-space-xs: 4px;
  --ui-space-sm: 8px;
  --ui-space-md: 12px;
```

色を直接繰り返さず、共通変数で管理する。必要な微調整は許可するが、暗い背景、明るい文字、明瞭な枠線という関係を維持する。

## 画面構成

### ヘッダー

- 左上にメニューボタンを置く。
- 現在の日付、画面名、実行中タイマーへ戻る導線を分かりやすくする。
- iPadのセーフエリアを考慮する。

### メイン画面

- 実行中タイマーがある場合は、タイマー表示を最優先にする。
- 通常時は、簡易時刻表と時刻未指定タスク一覧を上下に配置する。
- 完了・スキップ済み欄は画面下部に置き、折りたたみ可能にする。
- 重要な操作が画面外へ固定されたまま届かなくならないようにする。

### メニュー

- 左上のボタンから開く。
- 「今日」「検索」「履歴」「設定」「実行中タイマーへ戻る」などを表示する。
- 画面外にはみ出さず、必要ならメニュー内をスクロール可能にする。
- メニュー外タップ、閉じるボタン、Escapeキーで閉じられるようにする。

## パネルとカード

- 通常パネルは濃い背景と1px程度の枠線を使う。
- 現在実行中、終了待ち、入力フォーカスなど主役だけ2px程度の強調枠線を使う。
- 白い強調枠線をすべてに多用しない。
- 角丸は8pxから12px程度、内側余白は12pxから20px程度とする。
- 作業と休憩、未開始・実行中・一時停止・終了待ち・完了・スキップを、文字だけでなくラベルやアイコン形状でも区別する。

## ボタン

- タッチ領域は原則44px以上とする。
- ボタンの高さ、角丸、余白、文字サイズを統一する。
- 主操作はアクセント緑、危険操作は危険色、通常操作は暗い背景と枠線を使う。
- 無効状態を色だけでなく透明度や文言でも分かるようにする。
- hoverだけで意味を伝えない。
- タップ直後の押下状態を視覚的に示す。

## タイマー画面

- 残り時間を最も大きく表示する。
- 作業名、作業／休憩、状態、予定時間を近くに表示する。
- 「一時停止／再開」「完了」「スキップ」「5分延長」の優先度を明確にする。
- 終了待ちでは、通常タイマーと明確に異なる表示にし、超過時間を `+3分` のように示す。
- 誤タップしやすい「スキップ」と主操作を隣接させすぎない。

## 入力欄とダイアログ

- 入力欄は暗い背景、明瞭な枠線、十分な文字サイズを使う。
- ラベルをプレースホルダーだけに依存しない。
- iPadのソフトウェアキーボード表示中でも、保存・キャンセルへ到達できるようスクロールを確保する。
- モーダルは画面高さを超えないよう `max-height` と内部スクロールを持たせる。
- 戻しにくい操作は確認を表示する。
- 削除後の「元に戻す」は、画面下部など押しやすく邪魔になりにくい位置に表示する。

## レスポンシブ対応

- 固定幅だけに依存しない。
- `min()`、`max()`、`clamp()`、Flexbox、Gridを適切に使う。
- 狭い縦画面では、横並びを無理に維持せず上下へ積む。
- 横画面やPCで余白が増えても、操作が画面端へ離れすぎないよう最大幅を設定する。
- 不要な横スクロールを発生させない。
- `env(safe-area-inset-*)` を必要な箇所で考慮する。

## アクセシビリティ

- フォーカス表示を消さない。
- アイコンだけのボタンにはラベルまたはアクセシブル名を付ける。
- 色だけで状態を区別しない。
- 文字サイズを極端に小さくしない。
- PCのTab移動とEnter／Space操作を可能な範囲で確認する。
